# GoDriveLog Codex Guidance

Before planning or changing code or documentation, read:

1. `docs/Designs.md`
2. `docs/Implementation.md`
3. `docs/Status.md`

Documentation rules:

- Preserve design intent permanently.
- Keep design documents under `docs/Designs/`.
- Keep matching implementation records under `docs/Implementation/` using the same relative path.
- Treat `docs/Status.md` as the single source of truth for current progress.
- Do not add duplicate indexes or README files inside `docs/Designs/` or `docs/Implementation/`.
- Do not migrate or delete existing documentation without explicit scope and review.
