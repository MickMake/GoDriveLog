## Goal

Implement the shared `average` pointer marker engine.

`average` simulates an old-style highly damped physical pointer. It is not an arithmetic average, rolling mean, statistical calculation, or source-data average.

## Behaviour

The average pointer follows the final rendered indicator position using frame-rate-independent exponential smoothing with a fixed v3.6 time constant of 10 seconds:

```text
alpha = 1 - exp(-dt / 10s)
average_position = average_position + alpha * (rendered_indicator_position - average_position)
```

The 10 second time constant is intentionally not configurable in v3.6.

## Requirements

- Preserve prior v3.6 shared config/state/engine behaviour.
- Use `pointer_markers.average: true` as the enable flag.
- Preserve current rendering when `average` is absent or disabled.
- Initialise average marker state from the first valid final rendered indicator position.
- Update average marker state from final rendered indicator position only.
- Keep the implementation frame-rate independent.
- Keep the marker display-only; do not mutate source values, logs, exports, configured ranges, or input data.
- Add tests for default-disabled behaviour, fixed 10 second time constant, frame-rate independence, first-sample initialisation, and no arithmetic-average assumptions.

## Do not

- Do not calculate an arithmetic mean or rolling average.
- Do not make average damping configurable in v3.6.
- Do not implement radial or bar average rendering in this slice unless required by existing architecture as minimal plumbing.
- Do not add persistence.
- Do not add a `source` switch.
