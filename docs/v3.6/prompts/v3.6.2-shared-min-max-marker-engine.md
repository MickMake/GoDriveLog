# v3.6.2 Shared Min/Max Marker Engine

Implement the v3.6.2 slice only.

This file is one v3.6 implementation slice.

If invoked from a broad instruction such as "implement the next slice", first confirm this file matches the current target or the first unchecked allowed slice in `docs/v3.6/ImplementationState.md`.

Do not implement this slice if v3.6.1 is still unchecked.

## Goal

Implement shared min/max pointer marker tracking from the final rendered indicator position.

The marker engine should be gauge-family-neutral where practical. Radial and bar renderers can consume the resulting marker positions later.

## Input contract

Pointer markers sample the final rendered indicator position used by the gauge renderer.

They must not independently sample:

- source values;
- logs;
- exports;
- clean mapped values;
- pre-realism values.

Existing gauge logic remains responsible for value mapping and realism.

## History modes

### No `window`

No `window` means daily local reset mode.

- Use local dashboard/runtime time.
- Reset min/max at local midnight.
- Start unset until the first valid final rendered indicator position.
- Do not replay earlier history if the app starts partway through the day.

### With `window`

`window` means rolling min/max history.

- `window` must be positive and finite.
- Reject zero, negative, or unparseable windows.
- Timestamp retained samples.
- Discard samples older than the configured window.
- Recalculate min/max from remaining samples.
- Hide/unset a marker if no valid samples remain.

## Requirements

- Preserve v3.6.1 config/state behaviour.
- Track enabled min/max markers from final rendered indicator positions.
- Allow visible overshoot, bounce, stiction, or damping lag to affect min/max when those effects move the final rendered indicator.
- Keep state runtime-only; do not add database persistence.
- Keep marker state display-only; do not mutate source values, logs, exports, configured ranges, or input data.
- Add tests for daily reset, rolling expiry, min/max recalculation after expiry, no valid samples remaining, and rendered-position sampling.

## Do not

- Do not implement radial rendering in this slice.
- Do not implement bar rendering in this slice.
- Do not implement average marker damping in this slice.
- Do not ignore rendered overshoot or bounce in favour of a stable-only value.
- Do not add a `source` switch.
