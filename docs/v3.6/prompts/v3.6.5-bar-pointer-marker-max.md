## Goal

Add an optional bar gauge max pointer marker using the same pointer-following semantics proven by radial markers.

The bar max marker records the highest/furthest rendered bar indicator position seen since marker reset. It follows the rendered bar path/fill, not a separate source-value statistics stream.

## Config

Preferred config shape:

```yaml
realism:
  pointer_markers:
    max:
      enabled: true
```

The config key is shared with radial gauges, but rendering is gauge-family specific.

## Requirements

- Apply to `bar` gauges only in this slice.
- Preserve current bar rendering when `realism.pointer_markers.max` is absent or disabled.
- Preserve existing radial pointer marker behaviour.
- Add package/config support for bar `realism.pointer_markers.max`.
- Track the highest/furthest rendered bar position seen by the marker since reset.
- Respect existing bar orientation and direction rules.
- If bar realism effects are enabled, observe the resulting rendered bar behaviour.
- Keep marker state display-only; do not mutate source values, logs, exports, configured ranges, or input data.
- Use a bar-specific marker asset or render primitive. Do not reuse radial needle assumptions unless the existing renderer already abstracts marker geometry cleanly.
- Render the max marker in a predictable layer relative to the live bar fill/indicator and foreground overlays.
- Add tests for default-disabled behaviour, config parsing/validation, marker capture on higher/further bar movement, orientation handling, no mutation of input values, and render model output.
- Add a single-feature visual inspection fixture.
