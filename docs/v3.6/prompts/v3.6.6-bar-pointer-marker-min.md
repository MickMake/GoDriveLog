## Goal

Add an optional bar gauge min pointer marker using the same pointer-following semantics as the bar max marker.

The bar min marker records the lowest/least rendered bar indicator position seen since marker reset. It follows the rendered bar path/fill, not a separate source-value statistics stream.

## Config

Preferred config shape:

```yaml
realism:
  pointer_markers:
    min:
      enabled: true
```

The config key is shared with radial gauges, but rendering is gauge-family specific.

## Requirements

- Apply to `bar` gauges only in this slice.
- Preserve current bar rendering when `realism.pointer_markers.min` is absent or disabled.
- Preserve existing radial pointer marker behaviour.
- Preserve bar max marker behaviour from v3.6.5.
- Add package/config support for bar `realism.pointer_markers.min`.
- Track the lowest/least rendered bar position seen by the marker since reset.
- Respect existing bar orientation and direction rules.
- If bar realism effects are enabled, observe the resulting rendered bar behaviour.
- Keep marker state display-only; do not mutate source values, logs, exports, configured ranges, or input data.
- Use a bar-specific marker asset or render primitive. Do not reuse radial needle assumptions unless the existing renderer already abstracts marker geometry cleanly.
- Render min and max markers in a predictable layer relative to the live bar fill/indicator and foreground overlays.
- Add tests for default-disabled behaviour, min-only, min+max, lower/lesser bar capture, orientation handling, no mutation of input values, and render model output.
- Add a single-feature visual inspection fixture.

## Do not

- Do not alter radial marker behaviour.
- Do not implement damped marker in this slice.
- Do not implement mathematical average.
- Do not add persistence.
- Do not add a `source` switch.
