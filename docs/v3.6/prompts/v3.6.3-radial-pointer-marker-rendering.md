# v3.6.3 Radial Pointer Marker Rendering

Implement the v3.6.3 slice only.

This file is one v3.6 implementation slice.

If invoked from a broad instruction such as "implement the next slice", first confirm this file matches the current target or the first unchecked allowed slice in `docs/v3.6/ImplementationState.md`.

Do not implement this slice if v3.6.2 is still unchecked.

## Goal

Render radial min/max pointer markers using the shared min/max marker engine.

## Asset contract

Radial gauges use explicit marker needle PNG assets where provided:

```text
needle_min.png
needle_max.png
```

The assets use the same pivot/rotation geometry model as the live radial needle.

## Render order

Render radial pointer markers:

```text
background -> face -> ticks -> live needle -> pointer markers -> overlay/glass/bezel
```

Pointer markers sit above the live needle because traditional min/max drag markers were attached to, or visually associated with, the front glass/bezel assembly.

## Requirements

- Preserve v3.6.1 and v3.6.2 shared config/state/engine behaviour.
- Apply to radial gauges only in this slice.
- Preserve current radial rendering when pointer markers are absent or disabled.
- Render enabled min/max markers from shared marker positions.
- Use `needle_min.png` and `needle_max.png` where available.
- Use the same pivot/rotation model as the live radial needle.
- Render markers above the live needle and below overlay/glass/bezel layers.
- Keep marker assets visually distinct from the live needle.
- Add tests or render model checks for disabled behaviour, min-only, max-only, min+max, layer order, and pivot/rotation output.
- Add a radial visual inspection fixture showing min and max above the live needle.

## Do not

- Do not implement bar rendering in this slice.
- Do not implement average marker rendering in this slice.
- Do not replace explicit marker PNG assets with arbitrary procedural shapes.
- Do not alter existing live needle rendering semantics.
- Do not add persistence.
