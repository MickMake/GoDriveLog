## Goal

Add an optional radial max pointer marker.

The max marker records the highest rendered radial pointer position seen since marker reset. It follows the rendered needle path, not a separate source-value statistics stream.

If radial realism effects such as damping, overshoot, peg bounce, calibration offset, or later radial movement effects are enabled, the max marker observes the resulting rendered needle behaviour. If realism is disabled, the rendered needle path reflects the mapped source value.

## Config

Preferred config shape:

```yaml
realism:
  pointer_markers:
    max:
      enabled: true
```

A short boolean form may also be accepted if it fits the existing config parser cleanly:

```yaml
realism:
  pointer_markers:
    max: true
```

## Requirements

- Apply to `radial` gauges only in this slice.
- Preserve current radial rendering when `realism.pointer_markers.max` is absent or disabled.
- Add package/config support for radial `realism.pointer_markers.max`.
- Track the highest rendered radial needle position seen by the marker since reset.
- Allow overshoot or bounce to set the max marker if the rendered needle actually reaches that point.
- Keep marker state display-only; do not mutate source values, logs, exports, configured ranges, or input data.
- Use a radial marker asset or render model appropriate to the existing radial renderer. A default asset name such as `needle_max.png` is acceptable if it matches existing asset conventions.
- Use the same pivot/rotation geometry model as the live radial needle unless the existing renderer has a better marker convention.
- Render the max marker in a predictable layer relative to the live needle and foreground overlays.
- Add tests for default-disabled behaviour, config parsing/validation, marker capture on higher pointer movement, no mutation of input values, and render model output.
- Add a single-feature visual inspection fixture.
