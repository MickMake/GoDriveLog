## Goal

Add the shared pointer marker config and runtime state model used by radial and bar gauges.

This slice should establish the common spine before family-specific rendering is added.

## Config

Supported v3.6 shape:

```yaml
realism:
  pointer_markers:
    max: true
    min: true
    average: true
    window: 5m
```

Rules:

- `max`, `min`, and `average` are optional boolean enables.
- `window` is optional.
- Unknown keys under `pointer_markers` fail config loading clearly.
- Long-form object config is invalid.
- `pointer_markers: true` is invalid.

## Requirements

- Add shared config parsing/validation for `realism.pointer_markers`.
- Preserve existing gauge behaviour when `realism.pointer_markers` is absent.
- Preserve existing gauge behaviour when all pointer markers are disabled.
- Add shared runtime marker state structures suitable for radial and bar gauges.
- Keep marker state display-only; do not mutate source values, logs, exports, configured ranges, or input data.
- Do not render pointer markers yet except for any minimal internal scaffolding needed by existing architecture.
- Add tests for absent config, enabled markers, rejected long-form config, rejected unknown keys, rejected `pointer_markers: true`, and optional valid `window` parsing.

## Do not

- Do not implement radial or bar rendering in this slice.
- Do not implement min/max tracking logic beyond state scaffolding.
- Do not implement average damping in this slice.
- Do not add persistence.
- Do not add a `source` switch.
