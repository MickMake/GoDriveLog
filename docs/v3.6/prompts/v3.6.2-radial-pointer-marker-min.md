# v3.6.2 Radial Pointer Marker Min

Do not implement this slice if v3.6.1 is still unchecked.

## Goal

Add an optional radial min pointer marker using the same pointer-following semantics as the radial max marker.

The min marker records the lowest rendered radial pointer position seen since marker reset. It follows the rendered needle path, not a separate source-value statistics stream.

## Config

Preferred config shape:

```yaml
realism:
  pointer_markers:
    min:
      enabled: true
```

A short boolean form may also be accepted if it fits the existing config parser cleanly:

```yaml
realism:
  pointer_markers:
    min: true
```

## Requirements

- Apply to `radial` gauges only in this slice.
- Preserve current radial rendering when `realism.pointer_markers.min` is absent or disabled.
- Preserve the radial max marker behaviour from v3.6.1.
- Add package/config support for radial `realism.pointer_markers.min`.
- Track the lowest rendered radial needle position seen by the marker since reset.
- Allow overshoot or bounce to set the min marker if the rendered needle actually reaches that point.
- Keep marker state display-only; do not mutate source values, logs, exports, configured ranges, or input data.
- Use a radial marker asset or render model appropriate to the existing radial renderer. A default asset name such as `needle_min.png` is acceptable if it matches existing asset conventions.
- Use the same pivot/rotation geometry model as the live radial needle unless the existing renderer has a better marker convention.
- Render min and max markers in a predictable order relative to each other, the live needle, and foreground overlays.
- Add tests for default-disabled behaviour, min-only, min+max, lower pointer capture, no mutation of input values, and render model output.
- Add a single-feature visual inspection fixture.

## Do not

- Do not implement bar gauge markers in this slice.
- Do not implement damped secondary markers in this slice.
- Do not implement mathematical average.
- Do not add persistence.
- Do not add a `source` switch.
