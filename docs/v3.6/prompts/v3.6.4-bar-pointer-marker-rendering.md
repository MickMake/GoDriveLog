# v3.6.4 Bar Pointer Marker Rendering

Implement the v3.6.4 slice only.

This file is one v3.6 implementation slice.

If invoked from a broad instruction such as "implement the next slice", first confirm this file matches the current target or the first unchecked allowed slice in `docs/v3.6/ImplementationState.md`.

Do not implement this slice if v3.6.3 is still unchecked.

## Goal

Render bar min/max pointer markers using the shared min/max marker engine.

## Asset contract

Bar gauges use explicit marker PNG assets where provided:

```text
marker_min.png
marker_max.png
```

The assets are placed along the live bar fill/indicator axis.

## Render order

Render bar pointer markers:

```text
background/base -> bar track/face -> live bar/fill/indicator -> pointer markers -> overlay/frame/glass
```

Pointer markers sit above the live bar/fill because they are front-mounted visual markers, equivalent to the radial glass/bezel marker model.

## Requirements

- Preserve v3.6.1, v3.6.2, and v3.6.3 behaviour.
- Apply to bar gauges only in this slice.
- Preserve current bar rendering when pointer markers are absent or disabled.
- Render enabled min/max markers from shared marker positions.
- Use `marker_min.png` and `marker_max.png` where available.
- Respect existing bar horizontal, vertical, reversed, and origin/direction configuration.
- Render markers above the live bar/fill/indicator and below overlay/frame/glass layers.
- Keep marker assets visually distinct from the live bar/fill/indicator.
- Add tests or render model checks for disabled behaviour, min-only, max-only, min+max, orientation/direction handling, and layer order.
- Add a bar visual inspection fixture showing min and max above the live bar/fill/indicator.

## Do not

- Do not alter radial marker behaviour.
- Do not implement average marker rendering in this slice.
- Do not replace explicit marker PNG assets with arbitrary procedural shapes.
- Do not alter existing live bar/fill semantics.
- Do not add persistence.
