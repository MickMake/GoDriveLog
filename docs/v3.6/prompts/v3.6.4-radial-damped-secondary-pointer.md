## Goal

Add an optional radial damped secondary pointer marker.

This marker is inspired by heavily damped mechanical gauges that visually steady a fluctuating reading. It is not a mathematical average and must not be documented or tested as an arithmetic mean.

## Config

Preferred config shape:

```yaml
realism:
  pointer_markers:
    damped:
      enabled: true
```

Future options may tune damping, but keep this slice simple unless the existing renderer/config model already has a clear reusable damping parameter.

## Requirements

- Apply to `radial` gauges only in this slice.
- Preserve radial max/min marker behaviour.
- Preserve current radial rendering when `realism.pointer_markers.damped` is absent or disabled.
- Add package/config support for radial `realism.pointer_markers.damped`.
- Render a secondary radial pointer that moves more slowly/smoothly than the live pointer.
- Use a deterministic damping model compatible with existing radial movement code where possible.
- Keep the marker display-only; do not mutate source values, logs, exports, configured ranges, or input data.
- Use a radial marker asset or render model appropriate to the existing radial renderer. A default asset name such as `needle_damped.png` is acceptable if it matches existing asset conventions.
- Use the same pivot/rotation geometry model as the live radial needle unless the existing renderer has a better marker convention.
- Add tests for default-disabled behaviour, config parsing/validation, slow movement toward a changed value, no arithmetic-average assumptions, no mutation of input values, and render model output.
- Add a single-feature visual inspection fixture.

## Do not

- Do not call this feature `average` unless the surrounding docs already clearly frame it as a damped visual pointer rather than a mathematical average.
- Do not calculate arithmetic mean over samples.
- Do not implement bar damped marker in this slice.
- Do not add persistence.
- Do not add a `source` switch.
