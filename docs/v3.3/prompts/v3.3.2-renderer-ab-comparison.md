# v3.3.2 - renderer A/B comparison

## Goal

Compare Fyne and Ebiten using the same baseline dashboard, same upstream data path, and fixed-duration runs.

## Required context

Read:

- `docs/v3.3/ReleasePlan.md`
- `docs/v3.3/ImplementationState.md`
- `docs/v3.3/BaselineDashboardVerification.md`
- `examples/baseline-dashboard.yaml`

## Scope

- Ensure comparable renderer runs can use a fixed duration such as `--duration 60s`.
- Run Fyne and Ebiten with the same config, vehicle, pattern, interval, and duration.
- Capture display stats and Pi CPU observations.
- Record results in the v3.3 verification docs.

## Required command shape

```bash
go run ./cmd/GoDriveLog \
  --harness \
  --config ./examples/baseline-dashboard.yaml \
  --vehicle vw_caddy \
  --pattern sweep \
  --interval 50ms \
  --duration 60s \
  --renderer fyne
```

```bash
go run ./cmd/GoDriveLog \
  --harness \
  --config ./examples/baseline-dashboard.yaml \
  --vehicle vw_caddy \
  --pattern sweep \
  --interval 50ms \
  --duration 60s \
  --renderer ebiten
```

## Acceptance criteria

- Fyne and Ebiten are compared using the same baseline config.
- Both use the real dashboard runtime path.
- Results include `events`, `display_submitted`, `display_rendered`, `display_superseded`, and render duration fields.
- Pi smoothness and CPU observations are recorded where available.
- The result is decision-ready for v3.3.3.

## Do not

- Do not promote Ebiten in this slice.
- Do not change gauge package ownership rules.
- Do not treat desktop-only results as final.
- Do not rely on vibes unless the numbers are sitting beside them looking respectable.
