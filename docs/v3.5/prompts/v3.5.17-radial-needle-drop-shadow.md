# v3.5.17 Radial Needle Visual Polish

This file is one v3.5 implementation slice.

This slice may be implemented even while explicitly postponed earlier slices remain unchecked, but only when `docs/v3.5/ImplementationState.md` names v3.5.17 as the current target.

## Goal

Add two optional radial-only visual polish features:

- a radial needle drop shadow; and
- a display-only radial calibration offset.

These are deterministic radial rendering features. They are not an asset/layer system redesign, not dynamic parallax, and not general gauge physics.

## Requirements

- Apply to `radial` gauges only.
- Preserve current radial rendering when both features are absent or disabled.
- Add package/config support for an optional `realism.needle_shadow` setting.
- Add package/config support for an optional `realism.calibration_offset` setting in degrees.
- `calibration_offset` must default to zero.
- Apply `calibration_offset` after the normal value-to-angle calculation and before radial needle rendering.
- Keep `calibration_offset` display-only; it must not change input values, exported values, logs, configured ranges, or source data.
- Respect existing min/max angle limits or configured clamping behaviour when applying `calibration_offset`.
- The shadow should render as an offset, tinted/alpha copy of the same rotating needle geometry.
- The shadow must use the same pivot, scale, and final rendered needle angle as the real needle, including calibration offset.
- Render order should be: dial/background, needle shadow, real needle, existing foreground/overlay layers.
- Provide a simple configurable shadow offset, for example `[x, y]` pixels.
- Provide a simple configurable shadow alpha, clamped or validated to a sane range.
- Add tests for default-disabled/default-zero behaviour, config parsing/validation, render model output, positive/negative calibration offset, clamping/bounds, unchanged input value, and unchanged real needle value mapping before display offset.
- Add single-feature visual inspection fixtures for needle shadow and calibration offset.

## Do not

- Do not add blur in this slice.
- Do not add dynamic light direction, gyro/view-angle parallax, or moving shadows.
- Do not add idle animation or ambient effects.
- Do not change existing asset/layer semantics beyond optional radial shadow rendering.
- Do not apply either feature to non-radial gauge types.
- Do not alter source values or source value mapping.
- Do not implement stat markers in this slice.
