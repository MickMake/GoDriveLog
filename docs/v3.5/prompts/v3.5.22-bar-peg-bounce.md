## Goal

Add `realism.peg_bounce` support for bar gauges as end-stop bounce.

Although the config key remains `peg_bounce` for v3.5 consistency, bar gauges do not have a physical needle peg. For bars, this means a tiny bounded rebound of the displayed fill/reveal extent when the bar reaches its configured minimum or maximum visual stop.

## Requirements

- Implement `realism.peg_bounce` for bar gauges.
- Use the existing radial `peg_bounce` implementation as a reference for transition timing, lifecycle, clamping, and finite settle behaviour where sensible.
- Do not copy radial drawing or angle logic directly. Bars render fill/reveal extent, not needle angle.
- Trigger only at configured bar min/max visual stops.
- Keep the bounce small, deterministic, and bounded.
- Final settled extent must remain clamped to the target stop.
- Source values, logs, exported values, configured ranges, and input data must remain unchanged.
- Default/disabled behaviour must remain current/immediate bar rendering.
- Add tests for min stop, max stop, no-stop normal movement, disabled/default behaviour, final settlement, and unchanged source value.
- Add a single-feature visual inspection fixture for bar peg bounce / end-stop bounce.

## Do not

- Do not rename the public config key in this slice.
- Do not add generic `endstop_bounce` config in this slice.
- Do not change radial peg bounce in this slice unless required to share safe helper code.
- Do not add idle vibration or broader impact physics.
- Do not allow the bar to leak outside sensible visual bounds.
