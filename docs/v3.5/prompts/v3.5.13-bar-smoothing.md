# v3.5.13 Bar Smoothing

Implement the v3.5.13 slice only.

## Goal

Add finite smoothing for bar gauge value changes.

## Requirements

- Use the shared finite movement lifecycle.
- Support simple smoothing and optional different rise/fall timing if it fits existing config style.
- Default behaviour remains current/immediate bar rendering.
- Add tests for rising movement, falling movement, disabled/default behaviour, and final settled value.
- Add a single-feature visual inspection fixture.

## Do not

- Do not turn segmented gauges into bars.
- Do not add heartbeat or ambient pulse behaviour here.

Update `docs/v3.5/ImplementationState.md` to mark v3.5.13 complete and set the next target to v3.5.14.
