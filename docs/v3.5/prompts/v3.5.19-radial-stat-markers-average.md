# v3.5.19 Radial Stat Marker Average

This file is one v3.5 implementation slice.

If invoked from a broad instruction such as "implement the next slice", first confirm this file matches the current target or the next allowed slice in `docs/v3.5/ImplementationState.md`.

This slice may be implemented even while explicitly postponed earlier slices remain unchecked, but only when `docs/v3.5/ImplementationState.md` names v3.5.19 as the current target.

## Goal

Extend `realism.stat_markers` with an optional radial-only rolling-window average marker.

This slice builds on the v3.5.18 min/max stat marker implementation and reuses the same rolling-window semantics.

## Config

```yaml
realism:
  stat_markers:
    window: 1h
    average: true
```

`window` has the same meaning as v3.5.18:

- `window: 1h` means use stable displayed values from the last hour.
- `window: 24h` means use stable displayed values from the last day.
- `window: 0` means use all stable displayed values since runtime start.

## Requirements

- Apply to `radial` gauges only in this slice.
- Preserve existing min/max stat marker behaviour.
- Preserve current radial rendering when `realism.stat_markers` is absent or all markers are disabled.
- Add package/config support for `realism.stat_markers.average`.
- `average` renders `needle_average.png` at the rolling average stable displayed radial value inside the configured window.
- The average marker asset should use the same pivot/rotation geometry model as the live radial needle.
- Reuse the same stable displayed value history/window semantics introduced for min/max.
- Track stable displayed radial values after normal value mapping and `calibration_offset`.
- Do not capture temporary overshoot or peg-bounce excursions as average input samples.
- Keep average display-only. It must not mutate source values, logs, exported values, configured ranges, or input data.
- Render stat marker needles above the live needle and below existing foreground/overlay/bezel layers.
- Add tests for config parsing/validation, default-disabled behaviour, average-only, average with min/max, rolling-window expiry/recalculation, `window: 0`, render model output, and unchanged source/input values.
- Add a single-feature visual inspection fixture.

## Average semantics

Use a simple deterministic rolling average over the stable displayed values retained for the configured window.

If the existing runtime has a clear timestamped sample model, bound retained samples by timestamp. Do not store unbounded history.

If sample cadence questions arise, keep the implementation simple and document the chosen behaviour in `docs/v3.5/ImplementationState.md`.

## Do not

- Do not change min/max behaviour from v3.5.18.
- Do not implement bar gauge stat markers in this slice.
- Do not implement fade or decay behaviour.
- Do not implement `trip` or session-lifetime windows.
- Do not place stat markers under `movement`.
- Do not mutate source values, logs, exports, configured ranges, or input data.
