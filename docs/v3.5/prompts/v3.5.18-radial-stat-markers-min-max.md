## Goal

Add optional radial-only `realism.stat_markers` support for rolling-window minimum and maximum markers.

Stat markers display separate pointer assets at statistical values calculated over a trailing time window. They are display-only gauge markers, not source value changes.

This slice implements `min` and `max` only.

## Config

```yaml
realism:
  stat_markers:
    window: 1h
    min: true
    max: true
```

`window` defines the trailing time range used to calculate enabled stat markers.

- `window: 1h` means use stable displayed values from the last hour.
- `window: 24h` means use stable displayed values from the last day.
- `window: 0` means use all stable displayed values since runtime start.

## Requirements

- Apply to `radial` gauges only in this slice.
- Preserve current radial rendering when `realism.stat_markers` is absent or all markers are disabled.
- Add package/config support for `realism.stat_markers.window`.
- Add package/config support for `realism.stat_markers.min` and `realism.stat_markers.max`.
- `min` renders `needle_min.png` at the lowest stable displayed radial value inside the rolling window.
- `max` renders `needle_max.png` at the highest stable displayed radial value inside the rolling window.
- Both marker assets should use the same pivot/rotation geometry model as the live radial needle.
- Track stable displayed radial values after normal value mapping and `calibration_offset`.
- Do not capture temporary overshoot or peg-bounce excursions as stat marker values.
- When a new higher maximum enters the window, move the max marker to that value.
- When a new lower minimum enters the window, move the min marker to that value.
- When the currently displayed min/max sample leaves the rolling window, fall back to the next valid min/max sample still inside the window, or hide that marker if none exists.
- Keep marker history bounded by the configured rolling window.
- Keep stat markers display-only. They must not mutate source values, logs, exported values, configured ranges, or input data.
- Render stat marker needles above the live needle and below existing foreground/overlay/bezel layers.
- Add tests for config parsing/validation, default-disabled behaviour, min-only, max-only, min+max, rolling-window expiry/fallback, `window: 0`, render model output, and unchanged source/input values.
- Add a single-feature visual inspection fixture.

## Do not

- Do not implement `average` in this slice.
- Do not implement bar gauge stat markers in this slice.
- Do not implement fade or decay behaviour.
- Do not implement `trip` or session-lifetime windows.
- Do not use `needle_peak.png` for this feature.
- Do not place stat markers under `movement`.
- Do not mutate source values, logs, exports, configured ranges, or input data.
