## Goal

Add direction-dependent displayed offset for bar gauges without changing source values.

v3.5.16 covers radial display-only hysteresis. This slice adds the bar side separately so the implementation remains small and inspectable.

## Requirements

- Implement `realism.hysteresis` for bar gauges.
- Reuse the existing radial hysteresis implementation as a reference for direction tracking, config handling, display-only semantics, and tests where sensible.
- Do not copy radial angle logic directly. Bars render fill/reveal extent, not needle angle.
- Direction should affect only the displayed bar extent within configured bounds.
- Source values, logs, exported values, configured ranges, and input data must remain unchanged.
- Default/disabled behaviour must remain current/immediate bar rendering.
- Add tests for rising approach, falling approach, disabled/default behaviour, unchanged source value, and unsupported gauge types.
- Add a single-feature visual inspection fixture for bar hysteresis.

## Do not

- Do not apply hysteresis to raw sensor/source values.
- Do not change radial hysteresis in this slice unless required to share safe helper code.
- Do not add bar stiction or bar peg bounce in this slice.
- Do not add random drift or idle animation.
