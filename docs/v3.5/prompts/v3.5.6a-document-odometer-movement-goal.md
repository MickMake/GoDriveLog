# Codex Prompt: v3.5.6a document odometer movement goal

You are working in the `MickMake/GoDriveLog` repository.

Start from fresh `main`.

Task: create or update the v3.5 behaviour documentation for the odometer movement model.

## Context

The previous v3.5.6 implementation works, but the movement model is not the desired final design. The project needs an explicit behaviour contract before further odometer realism work.

The current direction with a separate `realism.movement_policy` knob is being replaced for odometers. Odometer movement should be controlled by one odometer-specific field.

`movement` should remain acceptable across gauge types for now. Gauge types without concrete movement behaviour may parse `movement` and keep their current immediate behaviour until their movement slice defines more.

## Goal

Document v3.5.6 as the odometer **main movement phase**.

The desired config shape is:

```yaml
odometer:
  movement: instant
```

Supported odometer movement values:

- `instant`: digit display jumps immediately to the target value. No animation.
- `linear`: wheel rolls from old digit position to target digit position at constant speed.
- `ease_out`: wheel starts moving fast, then slows into the target.
- `bell`: wheel starts slow, speeds up through the middle, then slows into the target. This is a bell-curve velocity / smoothstep-style movement.
- `smooth`: recognised only. Reserved for future enhancement. Warn and fall back to `instant` when implemented.
- `click`: recognised only. Reserved for future stepped-click enhancement. Warn and fall back to `instant` when implemented.

Do not use or recommend `realism.movement_policy` for odometers.

## Phase model

Document the odometer realism pipeline as:

```text
pre-movement modifiers -> main movement -> post-movement modifiers -> static offsets
```

For v3.5.6:

```text
none -> instant / linear / ease_out / bell -> none -> existing static offsets
```

Future slices should be able to compose around this:

- v3.5.2 `wraparound`: movement path rule.
- v3.5.3 `drum_slop`: static resting offset.
- v3.5.6 `movement`: main wheel movement phase.
- v3.5.7 `carry_drag` / 9-drag: pre/overlap movement on neighbouring wheels during rollover.
- v3.5.14 `snap_settle`: post-movement tail.
- v3.5.15 `backlash`: pre/post direction-change slack.

## Important rules

- v3.5.6 must not permanently feed fractional numeric odometer values back into the source display value.
- The movement phase should conceptually move from previous digit position to target digit position.
- At movement progress 1.0, the main movement handover position must be exactly the target digit position.
- Later post-movement effects such as `snap_settle` may start from that handover position.
- Existing source values, logs, exported values, ranges, and input data must not be mutated by realism.
