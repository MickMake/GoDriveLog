# v3.5.1 Gauge Preview Mode

This file is one v3.5 implementation slice.

If invoked from a broad instruction such as "implement the next slice", first confirm this file matches the first unchecked slice in `docs/v3.5/ImplementationState.md`.

Do not implement this slice if an earlier slice is still unchecked.

## Goal

Add an interactive Gauge Preview Mode for one gauge at a time.

This is not a test harness. It is a viewer for visually seeing what each realism option does.

## CLI

Add:

```text
godrivelog dashboard preview <file>
```

`<file>` is mandatory and must point to a normal dashboard/gauge YAML file.

Optional flags may include:

```text
--gauge <name>
--value <number>
--step <number>
--fine-step <number>
--coarse-step <number>
```

The CLI must not become a second realism configuration system. Realism options belong in YAML under `realism`.

## Required behaviour

- Keep the existing sweep/heartbeat behaviour.
- Add Gauge Preview Mode beside the existing options.
- Load one normal YAML gauge/dashboard preview file.
- Display one gauge.
- Start at the midpoint of the configured value range unless `--value` is supplied.
- Left arrow jumps to min.
- Right arrow jumps to max.
- Up arrow increments.
- Down arrow decrements.
- Shift + Up/Down uses a coarse step.
- Ctrl/Cmd + Up/Down uses a fine step.
- R resets to midpoint.
- Space replays the last transition where possible.
- Esc/Q quits.
- Mouse wheel may increment/decrement if simple to support.

## Preview file rules

- Use normal YAML preview files; do not add a special metadata layer.
- Add baseline and single-feature preview structure if useful.
- Permit one deliberate `99-all-options` preview file per gauge type.
- Numeric, segmented, and indicator gauges should only get baseline preview files unless a specific v3.5 realism option applies to them.

## Do not

- Do not add screenshots, videos, visual diff reports, generated art, or preview metadata.
- Do not implement any realism feature in this slice.
