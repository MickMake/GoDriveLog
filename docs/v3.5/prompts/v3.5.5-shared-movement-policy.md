# v3.5.5 Shared Movement Policy

Implement the v3.5.5 slice only.

## Goal

Add the first shared movement policy surface for finite transitions.

## Requirements

- Support simple policy names such as `immediate`, `linear`, and `ease_out`.
- Keep `immediate` equivalent to current behaviour.
- Keep policy state separate from source values.
- Validate unknown policy names clearly.
- Add tests for defaults, accepted policies, and invalid policies.

## Do not

- Do not add a broad easing catalogue.
- Do not add spring physics, idle animation, or gauge-specific effects.

## Completion

Update `docs/v3.5/ImplementationState.md` to mark v3.5.5 complete and set the next target to v3.5.6.
