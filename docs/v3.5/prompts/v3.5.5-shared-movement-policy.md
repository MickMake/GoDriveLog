# v3.5.5 Shared Movement Groundwork

Implement the v3.5.5 slice only.

This file is one v3.5 implementation slice. The filename is historical: this slice added shared finite-movement groundwork, but odometer movement no longer uses `realism.movement_policy` as the final model.

If invoked from a broad instruction such as "implement the next slice", first confirm this file matches the first unchecked slice in `docs/v3.5/ImplementationState.md`.

Do not implement this slice if an earlier slice is still unchecked.

## Goal

Add the first shared finite-movement lifecycle and helper surface needed by later gauge realism slices.

This slice may include simple transition helpers such as immediate, linear, and ease-out behaviour, but those helpers are implementation plumbing. They are not the final odometer configuration interface.

## Current doctrine

- `movement` is the single movement knob.
- Odometer wheel movement is configured with `odometer.movement`.
- Do not use or recommend `realism.movement_policy` for odometer movement.
- Keep movement state separate from source values.
- Validate unknown configured movement values clearly unless a gauge type explicitly documents a recognised fallback.

## Requirements

- Keep immediate/default behaviour equivalent to current behaviour.
- Keep movement state separate from source values.
- Add tests for defaults, accepted helper behaviour, and invalid configured movement values where applicable.
- Do not require odometers to use `realism.movement_policy`.

## Do not

- Do not add a broad easing catalogue.
- Do not add spring physics, idle animation, or gauge-specific effects.
- Do not make `realism.movement_policy` the public odometer movement API.
