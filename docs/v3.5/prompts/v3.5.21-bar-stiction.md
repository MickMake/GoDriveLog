## Goal

Add thresholded movement release for sticky bar fill/reveal extent.

## Requirements

- Implement `realism.stiction` for bar gauges.
- Reuse the existing radial stiction implementation as a reference for threshold handling, finite catch-up movement, lifecycle, and config handling where sensible.
- Do not copy radial drawing or angle logic directly. Bars render fill/reveal extent, not needle angle.
- Tiny value changes may hold the displayed bar extent until the configured release threshold is crossed.
- Once released, the displayed extent should catch up and settle cleanly.
- Large changes must not appear stuck or disconnected.
- Source values, logs, exported values, configured ranges, and input data must remain unchanged.
- Default/disabled behaviour must remain current/immediate bar rendering.
- Add tests for below-threshold movement, release after threshold crossing, large movement, disabled/default behaviour, final settlement, and unchanged source value.
- Add a single-feature visual inspection fixture for bar stiction.

## Do not

- Do not change radial stiction in this slice unless required to share safe helper code.
- Do not add bar peg bounce in this slice.
- Do not add random stickiness or idle animation.
- Do not allow stiction to accumulate permanent error.
