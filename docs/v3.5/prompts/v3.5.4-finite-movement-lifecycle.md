# v3.5.4 Finite Movement Lifecycle

Implement the v3.5.4 slice only.

## Goal

Add the shared lifecycle needed for finite value-change movement.

## Required model

The display should behave as:

```text
static -> value changed -> moving -> settled -> static
```

A tick or redraw loop should be requested only while movement is active.

## Requirements

- Store previous display value and target source value where needed.
- Make the lifecycle reusable by future gauge-specific movement features.
- Keep default behaviour equivalent to current static/immediate updates.
- Add tests for transition start, active state, completion, and no-op unchanged values.

## Do not

- Do not add easing variants beyond what is needed for lifecycle plumbing.
- Do not add perpetual animation.
- Do not add gauge-specific realism features in this slice.

Update `docs/v3.5/ImplementationState.md` to mark v3.5.4 complete and set the next target to v3.5.5.
