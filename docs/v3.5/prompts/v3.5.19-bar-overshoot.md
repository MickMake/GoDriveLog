## Goal

Add the missing bounded overshoot behaviour for bar gauges.

v3.5.10 was originally intended to cover radial and bar overshoot, but the implementation only covered radial behaviour. This slice adds the bar side without changing radial overshoot.

## Requirements

- Implement `realism.overshoot` for bar gauges.
- Reuse the existing radial overshoot implementation as a reference for transition timing, lifecycle, bounded pass-and-settle shape, and config handling where sensible.
- Do not copy radial drawing or angle logic directly. Bars render fill/reveal extent, not needle angle.
- The displayed bar extent may pass the target slightly, then return and settle exactly on the target extent.
- Keep overshoot small, deterministic, and bounded.
- Do not mutate source values, logs, exported values, configured ranges, or input data.
- Default/disabled behaviour must remain current/immediate bar rendering.
- Add tests for disabled/default behaviour, rising overshoot, falling overshoot, bounded overshoot, final settlement, and unchanged source value.
- Add a single-feature visual inspection fixture for bar overshoot.

## Do not

- Do not change radial overshoot in this slice unless required to share safe helper code.
- Do not add bar hysteresis, bar stiction, or bar peg bounce in this slice.
- Do not add general physics or repeated oscillation.
- Do not render outside sensible visual bounds.
