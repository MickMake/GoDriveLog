# v3.2.2 prompt: gauge widget support

## Goal

Add dashboard support for `type: gauge` widgets that place a loaded gauge package on a dashboard.

## Branch

Use a branch named:

```text
v3.2.2-gauge-widget-support
```

Branch from latest `main` after confirming the v3.2.1 PR is merged.

## Scope

Add the minimal dashboard config support needed to place a gauge package.

## Target dashboard shape

```yaml
widgets:
  - id: rpm
    type: gauge
    gauge: gauges/seven_segment/amber/4_digit
    position: [100, 20]
    scale: 1.0
```

## Behaviour

- Add `type: gauge` as a valid widget type.
- Add `gauge` path support to widget config.
- Add `scale` support if not already present.
- For `type: gauge`, require `gauge`.
- For `type: gauge`, do not require `asset`.
- For `type: gauge`, do not require widget-level `sensor`.
- Reject `sensor` on `type: gauge` widgets.
- Sensor binding comes from `gauge.yaml`.
- Existing widget types must keep their current validation rules.

## Out of scope

- Rendering the gauge.
- Creating scene parts.
- Fyne work.
- Example gauge package assets.
- Sensor override.
- Gauge inheritance.
- Clusters.

## Acceptance checks

- Config validation accepts a valid `type: gauge` widget.
- Config validation rejects a gauge widget missing `gauge`.
- Config validation rejects a gauge widget containing `sensor`.
- Config validation does not require `asset` for gauge widgets.
- Existing widget validation tests still pass.
- No existing widget type changes behaviour.
- `CHANGES.md` is updated.
- `docs/v3.2/ImplementationState.md` is updated.
