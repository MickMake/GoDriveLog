# v3.2.4 prompt: Fyne radial rendering

## Goal

Render radial gauge scene parts in the Fyne display adapter.

## Required reading

- `docs/v3.2/README.md`
- `docs/v3.2/ReleasePlan.md`
- `docs/v3.2/ImplementationState.md`
- `docs/v3.2/OpenDecisions.md`
- `docs/v3.2/CarryForward.md`
- `docs/v3.2/prompts/v3.2.3-radial-gauge-scene-model.md`

## Branch

Use a branch named:

```text
v3.2.4-fyne-radial-rendering
```

Branch from latest `main` after confirming the v3.2.3 PR is merged.

## Scope

Add Fyne adapter support for radial gauge scene parts.

## Behaviour

- Render layered PNG images in order: background, face, ticks, needle, overlay.
- Scale the gauge using the widget scale.
- Place the gauge using the widget position.
- Rotate the needle around its normalised `needle` pivot.
- Place the rotated needle pivot at the normalised `face` pivot on the gauge face.
- Preserve existing rendering for all existing widget types.
- Keep Fyne-specific image handling inside the display adapter.

## Out of scope

- Gauge package loading changes.
- Dashboard config changes.
- Scene model redesign beyond what is required by v3.2.3.
- Procedural drawing.
- Animation.
- Sensor overrides.
- Gauge inheritance.
- Clusters.

## Acceptance checks

- A radial gauge scene renders static layers.
- A radial gauge scene renders a rotated needle for valid live values.
- Needle pivot placement is visually and/or testably correct enough for the first pass.
- Non-`ok` states do not render fake live needle values.
- Existing widget rendering still works.
- `CHANGES.md` is updated.
- `docs/v3.2/ImplementationState.md` is updated.
- `docs/v3.2/OpenDecisions.md` is updated if Fyne rotation details are resolved.
