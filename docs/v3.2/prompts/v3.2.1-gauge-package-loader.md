## Scope

Add the minimal data model and loader support for gauge packages.

A gauge package is any directory under `assets/gauges/` containing a file named `gauge.yaml`.

Directory names under `assets/gauges/` carry no semantic meaning.

## Behaviour

The loader should:

- accept an asset-root relative gauge path such as `gauges/seven_segment/amber/4_digit`;
- resolve it to `assets/gauges/seven_segment/amber/4_digit/gauge.yaml`;
- parse the gauge YAML;
- support at least `type: seven_segment` and `type: radial` in parsed packages;
- preserve sensor binding from `sensor`;
- preserve package-owned fields such as `format`, `digit_set`, `digits`, `layers`, `pivot`, and `value_map` where applicable;
- resolve layer image paths relative to the `gauge.yaml` directory;
- allow relative paths like `../images/bezel.png` when they remain inside the asset root;
- reject path traversal outside the asset root;
- return clear validation/load errors.

## Minimal seven-segment gauge.yaml shape

```yaml
id: amber_4_digit_rpm
type: seven_segment
sensor: engine_rpm
format: "%04.0f"

size:
  width: 420
  height: 140

layers:
  panel: panel.png
  glass: glass.png

digit_set: amber_7seg

digits:
  count: 4
  positions:
    - [42, 35]
    - [132, 35]
    - [222, 35]
    - [312, 35]
```

## Minimal radial gauge.yaml shape

```yaml
id: rpm
type: radial
sensor: engine_rpm

size:
  width: 512
  height: 512

layers:
  background: ../images/bezel.png
  face: ../images/face_dark.png
  ticks: ticks.png
  needle: ../images/needle_red.png
  overlay: ../images/glass_overlay.png

pivot:
  face: { x: 0.5, y: 0.55 }
  needle: { x: 0.5, y: 0.9 }

value_map:
  min: 0
  max: 8000
  start_angle: -135
  end_angle: 135
  clamp: true
```
