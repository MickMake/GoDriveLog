# v3.2.5 prompt: example gauge package

## Goal

Add one small example gauge package that demonstrates the v3.2 gauge package layout.

## Required reading

- `docs/v3.2/README.md`
- `docs/v3.2/ReleasePlan.md`
- `docs/v3.2/ImplementationState.md`
- `docs/v3.2/OpenDecisions.md`
- `docs/v3.2/CarryForward.md`
- `docs/v3.2/prompts/v3.2.1-gauge-package-loader.md`
- `docs/v3.2/prompts/v3.2.2-gauge-widget-support.md`
- `docs/v3.2/prompts/v3.2.3-radial-gauge-scene-model.md`
- `docs/v3.2/prompts/v3.2.4-fyne-radial-rendering.md`

## Branch

Use a branch named:

```text
v3.2.5-example-gauge-package
```

Branch from latest `main` after confirming the v3.2.4 PR is merged.

## Scope

Add one working gauge package under `assets/gauges/`.

The directory names under `assets/gauges/` are arbitrary. The only required filename is `gauge.yaml`.

## Suggested shape

```text
assets/
  gauges/
    classic/
      images/
        bezel.png
        face_dark.png
        needle_red.png
        glass_overlay.png
      rpm/
        gauge.yaml
        ticks.png
```

## Behaviour

The example should:

- use `type: radial`;
- bind to an existing or testable RPM sensor such as `engine_rpm`;
- include realistic min/max range;
- include start/end angle mapping;
- use normalised face and needle pivots;
- demonstrate relative shared image paths if practical.

## Out of scope

- Multiple example gauges.
- New sensor semantics.
- Gauge inheritance.
- Clusters.
- Procedural drawing.
- Large binary asset dumps unless they are genuinely needed.

## Acceptance checks

- The example package loads.
- The example package can be referenced by a dashboard gauge widget.
- The example renders through the radial renderer if the previous slices are complete.
- Asset paths are relative and valid.
- `CHANGES.md` is updated.
- `docs/v3.2/ImplementationState.md` is updated.
- `docs/v3.2/OpenDecisions.md` is updated if the example asset source decision is resolved.
