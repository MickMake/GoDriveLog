## Scope

Add seven-segment gauge scene generation below the dashboard runtime and above the display adapter.

This slice proves the package model with a display type that is already conceptually understood.

## Behaviour

- Read sensor binding from the gauge package.
- Use current sensor state for that sensor.
- Read `format` from the gauge package, not from the widget.
- Use the configured digit count and digit positions from the gauge package.
- Reference the configured `digit_set` for reusable raw glyph artwork.
- Generate static layer scene data for panel/bezel and glass as available.
- Generate digit scene parts only when the sensor state is valid for live rendering.
- Include package size, widget position, widget scale, layer paths, digit positions, digit set, and formatted text in scene data.
- Include new seven-segment fields in scene signatures so scene changes are detected.

## Non-ok rule

For non-`ok` states, do not render fake live digit values.

Do not map missing, unsupported, timeout, parse_error, error, stale, or unknown to zeroes, blanks that look valid, or stale-looking live values unless a later explicit unavailable-state design is added.

## Out of scope

- Fyne drawing.
- Radial gauge scene model.
- Example assets.
- Sensor override.
- Gauge inheritance.
- Clusters.
- Performance optimisation beyond correct scene signatures.

## Acceptance checks

- Tests cover package-owned `format`.
- Tests cover 2, 3, 4, and 5 digit package shapes if practical.
- Tests cover digit positions coming from the package.
- Tests cover panel/glass/static layer scene data.
- Tests cover non-`ok` states not producing live digit values.
- Tests cover scene signature changes when formatted output changes.
- Existing dashboard scene tests still pass.
- `CHANGES.md` is updated.
- `docs/v3.2/ImplementationState.md` is updated.
- `docs/v3.2/OpenDecisions.md` is updated if the seven-segment scene structure decision is resolved.
