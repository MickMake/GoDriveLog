## Scope

Add radial gauge scene generation below the dashboard runtime and above the display adapter.

The dashboard runtime should consume sensor state and gauge package data, then emit scene data that the Fyne adapter can render later.

## Behaviour

- Read sensor binding from the gauge package.
- Use current sensor state for that sensor.
- For `ok` numeric values, map value to angle using `value_map`.
- Clamp when `value_map.clamp` is true.
- Generate static image layer parts for background, face, ticks, and overlay as available.
- Generate a needle part only when the sensor state is valid for live rendering.
- Include image paths, gauge size, widget position, widget scale, face pivot, needle pivot, and angle in the scene data.
- Include new radial fields in scene signatures so scene changes are detected.

## Non-ok rule

For non-`ok` states, do not render fake live gauge values.

Do not map missing, unsupported, timeout, parse_error, error, stale, or unknown to min, zero, or midpoint.

## Out of scope

- Fyne drawing.
- Image rotation implementation.
- Example assets.
- Sensor override.
- Gauge inheritance.
- Clusters.
- Performance optimisation beyond correct scene signatures.

## Acceptance checks

- Tests cover value-to-angle mapping.
- Tests cover clamping.
- Tests cover non-`ok` states not producing live needle values.
- Tests cover scene signature changes when radial angle changes.
- Existing dashboard scene tests still pass.
- `CHANGES.md` is updated.
- `docs/v3.2/ImplementationState.md` is updated.
- `docs/v3.2/OpenDecisions.md` is updated if the radial scene structure decision is resolved.
