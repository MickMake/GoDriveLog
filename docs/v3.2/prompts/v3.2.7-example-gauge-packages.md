## Branch

Use a branch named:

```text
v3.2.7-example-gauge-packages
```

Branch from latest `main` after confirming the v3.2.6 PR is merged.

## Scope

Add small working gauge packages under `assets/gauges/`.

The directory names under `assets/gauges/` are arbitrary. The only required filename is `gauge.yaml`.

## Suggested seven-segment shape

```text
assets/
  gauges/
    seven_segment/
      amber/
        2_digit/
          gauge.yaml
          panel.png
          glass.png
        3_digit/
          gauge.yaml
          panel.png
          glass.png
        4_digit/
          gauge.yaml
          panel.png
          glass.png
        5_digit/
          gauge.yaml
          panel.png
          glass.png
```

## Behaviour

The examples should:

- include at least one `type: seven_segment` package;
- use generated or placeholder PNG assets where needed;
- bind to an existing or testable sensor such as `engine_rpm`;
- keep `format` in `gauge.yaml`;
- demonstrate digit count and digit positions in `gauge.yaml`;
- include a radial example if radial rendering is complete enough to exercise;
- demonstrate relative shared image paths if practical.

## Out of scope

- Production-quality artwork.
- New sensor semantics.
- Gauge inheritance.
- Clusters.
- Procedural drawing.
- Large binary asset dumps unless genuinely needed.

## Acceptance checks

- Example packages load.
- Example packages can be referenced by dashboard gauge widgets.
- Seven-segment examples render through the seven-segment renderer.
- Radial example renders through the radial renderer if included.
- Asset paths are relative and valid.
- `CHANGES.md` is updated.
- `docs/v3.2/ImplementationState.md` is updated.
- `docs/v3.2/OpenDecisions.md` is updated if any example-specific decisions are resolved.
