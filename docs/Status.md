# GoDriveLog Documentation Status

Pillar 2 - The "current code truth".

This file is the single source of truth for the current state of documented features.

Design documents define intent. Implementation records explain how designs became code. This register records the currently verified implementation status for each documented feature.

## Verified implementation statuses

- **Implemented** - the documented design is present on `main` in the verified scope.
- **Partially implemented** - some of the design is present, but important parts remain absent or use a different contract.
- **Not implemented** - the design is still planned, historical, or otherwise absent from current code.
- **Unable to verify** - the design may exist, but this audit could not verify it from code and tests alone.
