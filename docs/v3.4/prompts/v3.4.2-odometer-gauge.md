# v3.4.2 - odometer gauge

## Goal

Add the first odometer gauge config and scene model.

## Required reading

1. `docs/v3.4/ReleasePlan.md`
2. `docs/v3.4/ImplementationState.md`
3. `docs/v3.4/BaselineDashboardVerification.md`

## Behaviour model

```text
value -> digit wheels -> strip offsets -> clipped rolling strips
```

`odometer` is a transform gauge. It uses the current value to calculate strip offsets for digit/wheel assets.

## Movement modes

```yaml
odometer:
  movement: smooth
```

`movement` defaults to `smooth`.

```text
smooth = continuous strip offset between digit positions
click  = stepped mechanical movement that snaps to digit positions
```

Keep this deliberately simple. Do not add easing, inertia, gear backlash, or animation curves in this slice.

## Scope

- Add `type: odometer` package validation.
- Add odometer package fields for wheels, strip assets, window positions, optional alignment offsets, optional wheel roles, and optional `movement`.
- Add scene parts needed for rolling wheel strips and overlays.
- Start with flat strip rendering.
- Support `movement: smooth` as the default.
- Support `movement: click` as the simple mechanical stepped option.
- Support a red/sub-unit wheel role without arbitrary decimal formatting.

## Out of scope

- Do not add curved wheel depth.
- Do not show rear-wheel wraparound yet.
- Do not implement a text-based odometer.
- Do not reuse `numeric` for odometer behaviour.
- Do not add easing, inertia, gear backlash, or physics-style animation.

## Done when

- A minimal odometer package can be loaded and rendered through the normal dashboard scene path.
- Smooth wheel movement is represented clearly in scene data or renderer behaviour.
- Click wheel movement snaps clearly to digit positions.
- Docs and implementation state are updated.
