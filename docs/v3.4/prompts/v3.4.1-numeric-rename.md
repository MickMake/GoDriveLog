# v3.4.1 - numeric rename

## Goal

Hard-rename the old `seven_segment` gauge type to `numeric`.

## Required reading

1. `docs/v3.4/ReleasePlan.md`
2. `docs/v3.4/ImplementationState.md`
3. `docs/v3.4/BaselineDashboardVerification.md`

## Scope

- Rename code constants and user-facing type checks from `seven_segment` to `numeric`.
- Update gauge package examples from `type: seven_segment` to `type: numeric`.
- Update baseline verification docs after the runnable baseline works.
- Keep the existing numeric rendering behaviour: formatted value, character slots, decimal point handling, and image assets.

## Out of scope

- Do not add a compatibility alias.
- Do not add a `style` field.
- Do not add Nixie/LCD/VFD/dot-matrix special cases in code.
- Do not implement odometer, indicator, bar, or segmented gauges here.

## Done when

- Existing baseline numeric displays run as `type: numeric`.
- No active examples still use `type: seven_segment`.
- Tests and docs are updated.
- `CHANGES.md` and `docs/v3.4/ImplementationState.md` are updated.
