# v3.4.8 - neon-grid dashboard

## Goal

Add the generated neon-grid dashboard example.

This dashboard should be dark retro tech with a Tron-like feel: neon blue first, cyan/white second, and just enough glow to make the dashboard look electrically smug.

## Theme name

Use:

```text
neon-grid
```

## Visual direction

Use a dark retro-tech look:

- near-black dashboard background;
- neon blue as the primary colour;
- cyan/white secondary highlights;
- subtle grid or circuit-line background accents;
- glowing radial tick marks and arcs;
- bright cyan/white needle treatment;
- luminous blue numeric displays;
- glowing indicator tiles;
- bar and segmented gauges that look like energy strips or stepped light bands.

Do not over-glow everything. If every pixel is shouting, the dashboard becomes a gaming keyboard in a thunderstorm.

## Scope

Add only the neon-grid themed dashboard and its generated assets.

Expected output may include:

```text
examples/assets/v3.4/neon-grid/
examples/dashboards/neon-grid.yaml
```

Use the generation framework from v3.4.6. Extend the framework only where needed for this theme.

## Gauge coverage target

Cover as much of the completed v3.4 gauge model as practical:

- `radial` RPM or speed gauge with blue glowing ticks/arcs;
- `numeric` display using luminous blue/cyan assets;
- `odometer` with a digital/tech strip treatment;
- `indicator` glowing icon tiles;
- `bar` continuous fuel/temperature/energy strip;
- `segmented` stepped neon threshold display.

Do not create new gauge behaviour to achieve the look.

## Asset rules

- Generate all assets deterministically.
- Keep transparent glow layers where useful.
- Use procedural glow/blur approximations that remain deterministic.
- Keep generated source/config committed.
- Do not use remote image generation.
- Do not download stock textures or icons.
- Do not hand-edit generated PNGs as the source of truth.

## Runtime rules

- Use existing v3.4 gauge types only.
- Do not add runtime `style` fields.
- Do not alter renderer behaviour unless fixing a genuine example-loading bug.
- Keep grid lines, circuit traces, glows, tiles, and decorative tech details as image assets.

## Validation

Run the generator.

Confirm:

- neon-grid assets are generated;
- the example dashboard config references the generated assets correctly;
- existing tests still pass or known existing failures are documented;
- any available dashboard/example validation passes;
- a preview screenshot is generated only if an existing repo mechanism supports it.

- `examples/dashboards/neon-grid.yaml` or equivalent exists.
- The dashboard demonstrates the dark neon visual direction without new renderer behaviour.
