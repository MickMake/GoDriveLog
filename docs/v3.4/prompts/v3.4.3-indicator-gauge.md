# v3.4.3 - indicator gauge

## Goal

Add a simple two-state indicator gauge.

## Required reading

1. `docs/v3.4/ReleasePlan.md`
2. `docs/v3.4/ImplementationState.md`
3. `docs/v3.4/BaselineDashboardVerification.md`

## Behaviour model

```text
sensor/state -> off image OR on image
```

## Scope

- Add `type: indicator` package validation.
- Add renderer/scene support for selecting an `off` or `on` layer.
- Keep layers simple: bezel, face, off, on, glass.
- Define the first truth rule clearly in package config or sensor state handling.

## Out of scope

- Do not add multi-state warning panels.
- Do not add blinking/animation in the first slice unless it is almost free.
- Do not turn this into a general glyph/text warning display.

## Done when

- An indicator gauge package can show off/on through the normal dashboard path.
- Missing state assets fail clearly.
- Docs and implementation state are updated.
