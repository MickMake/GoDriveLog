# v3.4.5 - segmented gauge

## Goal

Add sparse percent-threshold image selection for `type: segmented`.

## Required reading

1. `docs/v3.4/ReleasePlan.md`
2. `docs/v3.4/ImplementationState.md`
3. `docs/v3.4/BaselineDashboardVerification.md`

## Behaviour model

```text
value -> normalized/clamped percent -> threshold selection -> image layer
```

`segmented` is an image-selection gauge. It does not calculate continuous transform geometry like `radial`, `odometer`, or `bar`.

## Layer pattern

```yaml
layers:
  segments: levels/rpm_{percent:03}.png
```

Example files:

```text
rpm_000.png
rpm_010.png
rpm_030.png
rpm_040.png
```

These are valid sparse thresholds. The filename percent is not a dense index.

## Hysteresis

```yaml
segmented:
  hysteresis: 25
```

`hysteresis` defaults to `25`.

The value is a percentage of the adjacent threshold gap, not an absolute percentage of the full `0..100` range.

Example:

```text
thresholds: 25, 50
hysteresis: 25
falling from 50 uses (50 - 25) * 25% = 6.25
050 remains active until value drops below 43.75
```

This keeps threshold flicker under control while staying relative to sparse segment spacing.

## Scope

- Add `type: segmented` package validation.
- Detect `{percent}` in the segments layer path.
- Discover matching files by scanning the target directory.
- Extract numeric percent thresholds.
- Sort percent thresholds.
- Normalize and clamp runtime values to `0..100` with no config option.
- Select the highest threshold reached by the current normalized percent.
- Apply threshold-gap hysteresis when falling back to a lower threshold.
- Lazy-load selected images.
- Allow missing `000`.
- Display no segmented value layer below the first discovered threshold when `000` is missing.
- Ignore non-matching files.
- Ignore files above `100` and print a warning.

## Single-threshold behaviour

A single valid file is allowed:

```text
indicate_050.png
```

That behaves as a value-driven overlay:

```text
value < 50  -> display no segmented value layer
value >= 50 -> display indicate_050.png
```

This is different from `indicator`, which chooses off/on from incoming state rather than normalized value thresholds.

## Out of scope

- Contiguous files are not required.
- A `steps` config field is not required.
- All threshold images should not be decoded at startup.
- `bar` remains separate.
- Per-segment geometry/layout handling is deferred.
- Continuous transform behaviour belongs in `bar`, not `segmented`.

## Done when

- Sparse percent threshold files render correctly.
- Missing `000` renders no segmented value layer below the first threshold.
- Threshold-gap hysteresis prevents flicker around threshold boundaries.
- Missing or malformed patterns fail clearly.
- Runtime does not decode all matching images at startup.
- Docs and implementation state are updated.
