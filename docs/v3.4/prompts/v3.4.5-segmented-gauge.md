# v3.4.5 - segmented gauge

## Goal

Add sparse percent-threshold image selection for `type: segmented`.

## Required reading

1. `docs/v3.4/ReleasePlan.md`
2. `docs/v3.4/ImplementationState.md`
3. `docs/v3.4/BaselineDashboardVerification.md`

## Behaviour model

```text
value -> normalized percent -> choose highest discovered percent image reached
```

## Layer pattern

```yaml
layers:
  segments: levels/rpm_{percent:03}.png
```

Example files:

```text
rpm_000.png
rpm_010.png
rpm_030.png
rpm_040.png
```

These are valid sparse thresholds. The filename percent is not a dense index.

## Scope

- Add `type: segmented` package validation.
- Detect `{percent}` in the segments layer path.
- Discover matching files by scanning the target directory.
- Extract numeric percent thresholds.
- Sort percent thresholds.
- Select the highest threshold less than or equal to the current normalized percent.
- Lazy-load selected images.

## Out of scope

- Contiguous files are not required.
- A `steps` config field is not required.
- All threshold images should not be decoded at startup.
- `bar` remains separate.
- Per-segment geometry/layout handling is deferred.

## Done when

- Sparse percent threshold files render correctly.
- Missing or malformed patterns fail clearly.
- Runtime does not decode all matching images at startup.
- Docs and implementation state are updated.
