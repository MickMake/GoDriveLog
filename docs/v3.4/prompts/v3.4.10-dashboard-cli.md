## Defaults and config resolution

`--config` is optional for commands that accept it.

When a positional config file or `--config` is supplied, load exactly that file and bypass config discovery.

If both a positional config file and `--config` are supplied to `dashboard validate`, return an error.

Do not default to `config.example.yaml`.

Candidate config filenames are:

```text
godrivelog.yaml
godrivelog.yml
dashboard.yaml
dashboard.yml
config.yaml
config.yml
```

When no config file is supplied, search config files from:

```text
current working directory
/etc/godrivelog recursively
```

Directory traversal must be deterministic:

- sort each directory's entries alphabetically before evaluating them;
- search the current working directory non-recursively;
- search `/etc/godrivelog` recursively;
- evaluate only candidate config filenames.

### No vehicle specified

For every candidate config file in search order:

- if the config defines exactly one vehicle, use that config and that vehicle;
- if the config defines multiple vehicles, stop and return an error requiring a vehicle ID.

Do not continue past the first valid multi-vehicle config when no vehicle ID was supplied.

The multi-vehicle error must include the config path and available vehicle IDs.

### Vehicle specified

For every candidate config file in search order:

- if the config defines the requested vehicle ID, use that config and that vehicle;
- otherwise continue searching.

The matching vehicle may be in a single-vehicle or multi-vehicle config.

If no searched config contains the requested vehicle ID, return a useful error listing the searched config files and the vehicle IDs found in each valid config.

### Explicit config plus vehicle

When an explicit config file is supplied and a vehicle ID is supplied, do not continue searching other files. Load that config only. If it does not contain the requested vehicle ID, return an error for that file.

## Validate command

`dashboard validate` validates the resolved config using existing config validation behaviour.

Allowed forms:

```text
GoDriveLog dashboard validate
GoDriveLog dashboard validate ./config.yaml
GoDriveLog dashboard validate --config ./config.yaml
```

Rules:

- If no config is supplied, use normal config discovery.
- If a positional config is supplied, validate that file.
- If `--config` is supplied, validate that file.
- If both positional config and `--config` are supplied, return an error.
