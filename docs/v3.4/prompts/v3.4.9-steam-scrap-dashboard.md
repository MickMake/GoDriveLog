# v3.4.9 - steam-scrap dashboard

## Goal

Add the generated steam-scrap dashboard example.

This dashboard should look steampunk, overbuilt, and assembled from recycled mechanical and electrical leftovers. It should have unnecessary pipes, dials, bits of wire, rivets, plates, lamps, and the general confidence of a machine that requires both a spanner and a prayer.

## Theme name

Use:

```text
steam-scrap
```

## Visual direction

Use a steampunk/scrapyard look:

- tarnished brass, copper, dark iron, and aged cream faces;
- mismatched metal plates;
- unnecessary pipes and fittings as background/overlay decoration;
- wire loops, cable runs, rivets, bolts, screws, and brackets;
- brass or oxidised metal bezels;
- dark or red-painted scrap-metal needles;
- nixie-ish orange numeric display treatment where useful;
- small lamps or caged indicators;
- deliberately overbuilt details that still leave the gauges readable.

It should look recycled, practical, and a little ridiculous. If it starts looking like a clean luxury watch, rough it up.

## Scope

Add only the steam-scrap themed dashboard and its generated assets.

Expected output may include:

```text
examples/steam-scrap/dashboard.yaml
examples/steam-scrap/assets/
```

Use the generation framework from v3.4.6. Extend the framework only where needed for this theme.

## Gauge coverage target

Cover as much of the completed v3.4 gauge model as practical:

- `radial` RPM or speed gauge with brass/aged face treatment;
- `numeric` display with nixie-ish orange or amber assets;
- `odometer` with mechanical drum/scrap-metal frame treatment;
- `indicator` small lamp/caged warning light treatment;
- `bar` pressure-tube or slot level gauge treatment;
- `segmented` stepped brass/indicator-plate display.

Do not create new gauge behaviour to achieve the look.

## Asset rules

- Generate all assets deterministically.
- Keep transparent decorative overlays where useful.
- Use procedural tarnish, patina, scratches, rivets, and plate seams.
- Keep generated source/config committed.
- Do not use remote image generation.
- Do not download stock metal textures or icons.
- Do not hand-edit generated PNGs as the source of truth.

## Runtime rules

- Use existing v3.4 gauge types only.
- Do not add runtime `style` fields.
- Do not alter renderer behaviour unless fixing a genuine example-loading bug.
- Keep pipes, wires, plates, rivets, screws, lamps, and decorative clutter as image assets.
- Do not make the renderer understand decorative plumbing. That is how dashboards become boilers.
