# v3.4.4 - bar gauge

## Goal

Add the first continuous bar gauge behaviour.

## Required reading

1. `docs/v3.4/ReleasePlan.md`
2. `docs/v3.4/ImplementationState.md`
3. `docs/v3.4/BaselineDashboardVerification.md`

## Behaviour model

```text
value -> normalized percent -> continuous fill/reveal/movement
```

## Scope

- Add `type: bar` package validation.
- Add first mode for clipping/revealing a `level` layer within bounds.
- Keep visual identity asset-only.
- Keep config focused on placement behaviour, not visual style.

## Suggested first config shape

```yaml
bar:
  mode: level
  axis: vertical
  origin: bottom
  bounds: [40, 20, 24, 180]
```

## Out of scope

- Do not implement segmented percent image selection here.
- Do not implement many bar modes at once.
- Do not add style names.

## Done when

- A bar gauge can continuously reveal a level image from a normalized sensor value.
- Renderer behaviour is tested through the normal dashboard path.
- Docs and implementation state are updated.
