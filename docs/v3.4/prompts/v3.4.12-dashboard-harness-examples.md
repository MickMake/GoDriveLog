## Harness command

`dashboard harness` runs the dashboard harness for the selected vehicle through the dashboard command tree.

Keep existing pattern options for now. `sweep` remains the default.

For this slice, `sweep` becomes gauge-aware synthetic input. It should drive each gauge in a way that matches the gauge behaviour instead of treating every gauge as the same numeric sweep.

Gauge-aware sweep behaviour:

| Gauge type | Sweep behaviour |
|---|---|
| `odometer` | Let `n = 0`; increment from `n - 20` to `n + 20` for 5 seconds, then `n + 20` to `n + 30` for 5 seconds. |
| `numeric` | Same as `odometer`. |
| `radial` | Keep the existing sweep style. |
| `indicator` | Flash on/off for 5 seconds with a 1s duty cycle, then flash on/off for 5 seconds with a 250ms duty cycle. |
| `bar` | Heartbeat pulse at 90 bpm. |
| `segmented` | Same input shape as `radial`. |

Do not over-expand `heartbeat` or `fixed` yet. If they look odd for some gauge types, leave that for a later scope decision.

## Examples command

`dashboard examples` generates a deterministic example dashboard.

`--output <directory>` is required.

The output path is the dashboard root directory:

```text
<output>/
  dashboard.yaml
  assets/
    panel/
    gauges/
```

Rules:

- create `<output>` if it does not exist;
- write `dashboard.yaml` directly under `<output>`;
- write `assets/` directly under `<output>`;
- do not create a theme-named subdirectory under `<output>`;
- all internal dashboard and gauge-package paths must be relative to `<output>` so the generated dashboard directory can be moved anywhere;
- if `<output>` exists and is empty, use it;
- if `<output>` exists and is non-empty, interactive terminals may prompt unless `--force` is supplied;
- if `<output>` exists and is non-empty in non-interactive use, fail unless `--force` is supplied;
- if `<output>` exists and is a file, return an error.

Keep generated example output deterministic and local. Do not use remote image generation, stock art downloads, or hand-edited opaque PNGs as source of truth.

The implementation may reuse or wrap the existing generated example asset machinery, but the public CLI shape should remain `dashboard examples`.
