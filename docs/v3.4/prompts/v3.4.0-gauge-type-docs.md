# v3.4.0 - gauge type docs

## Goal

Create the v3.4 planning docs and implementation notes for gauge/display package cleanup and expansion.

## Required reading

1. `docs/v3.4/ReleasePlan.md`
2. `docs/v3.4/ImplementationState.md`
3. `docs/v3.4/BaselineDashboardVerification.md`

## Scope

- Create `docs/v3.4/`.
- Mirror the v3.3 release doc filenames where appropriate.
- Create `docs/v3.4/prompts/`.
- Add one file per planned v3.4 implementation slice.
- Capture the final gauge type model: `numeric`, `radial`, `odometer`, `indicator`, `bar`, and `segmented`.
- Record that visual identity is asset-only.
- Record that `seven_segment` will be hard-renamed to `numeric`.
- Record that `segmented` uses `{percent}` sparse threshold images.

## Out of scope

- Do not implement code changes.
- Do not rename current gauge package YAML yet.
- Do not add compatibility aliases.
- Do not add a `style` field.

## Done when

- v3.4 docs exist.
- v3.4 slice files exist.
- Implementation state points to the next slice.
- A PR is opened for review.
